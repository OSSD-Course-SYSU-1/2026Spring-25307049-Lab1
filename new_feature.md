# 作业要求 3：新增功能描述文档

## 1. 功能名称
**基于 @ComponentV2 的实时搜索过滤系统**

## 2. 实现原理
- **状态监听**：利用新框架的 `@Local` 装饰器定义 `searchText`，实现 UI 与搜索词的双向绑定。
- **动态筛选**：自定义 `getFilteredData()` 函数。由于原始数据 `COMPONENT_DATA` 是嵌套结构（组 > 项），该函数通过两层 `filter` 和 `map` 逻辑，在用户输入时动态重组显示列表。
- **性能优化**：通过 `trim()` 和 `toLowerCase()` 处理输入，确保搜索的鲁棒性，并适配了 ArkTS 的严格类型校验。

## 3. 用户价值
在组件库日益庞大时，用户无需手动翻找，通过关键词即可秒级定位到目标 UI 组件示例。
